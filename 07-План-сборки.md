# План сборки сайта In-CRM

> **Для исполнителя:** проект — не обычный код с юнит-тестами, а многостраничный сайт-прототип. «Тест-цикл» каждой задачи = **визуальная и структурная проверка** страницы (рендер, адаптив 375/768/1280, доступность, отсутствие ошибок в консоли, SEO-разметка), а не pytest. Отметки — чекбоксы `- [ ]`.

**Цель:** собрать многостраничный сайт In-CRM в редакторском бордо-языке по [06-Спека-сайта.md](06-Спека-сайта.md), с прицелом на SEO.

**Архитектура:** статические HTML-страницы (SEO-дружелюбный статический вывод) + один общий CSS дизайн-системы (`assets/site.css`). Сквозной каркас (шапка/подвал/блок аудита/крошки) собирается единообразно на каждой странице. Без сборщика на этапе прототипа — генератор/CMS остаются отдельным решением для продакшена (спека §5).

**Стек:** HTML5 (семантика), CSS (переменные, grid/flex), минимальный vanilla JS (бургер, аккордеоны, демо-формы), Schema.org JSON-LD, системные шрифты. Локальный просмотр — `static-python` :8000 (`.claude/launch.json`).

## Глобальные ограничения (из спеки + договорённости)

- Палитра: `ink #17150F`, `paper #F3EFE6`, `panel #ECE6D9`, `muted #6E675A`, `line rgba(23,21,15,.16)`, **АКЦЕНТ бордо `#7E2434`**. Один осознанный light-«бумажный» режим.
- Типографика: display — плотный гротеск (системный стек); voice — моно (Cascadia/Consolas); body — системный sans; `tabular-nums` для цифр.
- Каждая страница: **один H1**, семантическая иерархия, мета `title`/`description`, Open Graph, релевантная Schema.org, хлебные крошки (кроме Главной).
- Доступность: `:focus-visible`, `prefers-reduced-motion`, контраст; адаптив десктоп/планшет/моб + бургер.
- ЧПУ по спеке §1. Данные результатов/отзывов/лого/фото — плейсхолдеры, помечать как placeholder.
- **Код-стиль:** комментарии в коде пишутся **на русском языке**.

## Структура файлов

- `assets/site.css` — дизайн-система (токены, типографика, компоненты, утилиты, адаптив)
- `assets/site.js` — общий JS (бургер, аккордеон, демо-формы)
- `index.html` — Главная
- `uslugi/index.html` — хаб + `uslugi/{vnedrenie-amocrm,integracii,vidzhety,obuchenie,soprovozhdenie,analitika}.html` (общий шаблон услуги)
- `keysy/index.html` — список + `keysy/<slug>.html` (шаблон кейса)
- `tarify.html`, `o-kompanii.html`, `kontakty.html`
- `otrasli/index.html` — хаб + `otrasli/<slug>.html` (шаблон отрасли)
- `blog/index.html` — список + `blog/<slug>.html` (шаблон статьи)
- `sitemap.xml`, `robots.txt` (Фаза 3)

## Фаза 1 — Дизайн-система + каркас + Главная

### Задача 1: Дизайн-система (`assets/site.css`) + витрина компонентов
- **Files:** create `assets/site.css`, create `_styleguide.html` (временная витрина).
- **Содержание:** `:root` токены; reset; типо-шкала (display / h1–h4 / body / label / mono); кнопки (`.btn` ink/accent/ghost, `.link`); карточки (`.card` + модификаторы услуга/кейс/статья/тариф/отзыв/команда); `.index` (нумерованный), `.stats` (строка цифр), `.faq` (аккордеон), `.field` (формы), `.breadcrumbs`, `.audit`; сетки/утилиты; `:focus-visible`; `@media` брейкпоинты; `prefers-reduced-motion`.
- [ ] Собрать `site.css` + витрину всех компонентов на `_styleguide.html`
- [ ] Проверка: открыть через `static-python`, все компоненты рендерятся, бордо-акцент на месте, консоль чистая
- [ ] Проверка адаптива (375 / 768 / 1280)
- [ ] **Чек-поинт:** показать витрину заказчику (акцент, типографика, компоненты) до сборки страниц

### Задача 2: Сквозной каркас (шапка, подвал, блок аудита, крошки)
- **Files:** эталонная разметка блоков в `_styleguide.html`, стили в `site.css`, поведение в `site.js`.
- **Содержание:** шапка (лого IN—CRM, меню по карте сайта, телефон, CTA «Аудит», бургер на моб); подвал (колонки услуги/компания/контакты, реквизиты, мессенджеры); блок «Бесплатный аудит воронки» (заголовок, буллеты, демо-форма имя+телефон); хлебные крошки.
- [ ] Собрать блоки + `site.js` (бургер)
- [ ] Проверка: рендер, бургер кликается, демо-форма не сабмитит реально, консоль чистая, адаптив

### Задача 3: Главная (`index.html`)
- **Files:** create `index.html`.
- **Секции:** герой-тезис «Порядок в продажах. За 30 дней.» + CTA → строка цифр (45+/17/30дн/6лет) → ключевые услуги (сетка → `/uslugi/`) → «как внедряем за 30 дней» (4 недели, нумерация) → превью кейсов (2–3 «было→стало») → превью тарифов (3 → `/tarify/`) → команда → отзывы → FAQ → блок аудита → подвал.
- **SEO:** `<title>`/`<meta desc>`, OG, JSON-LD Organization + LocalBusiness (Калининград, телефон), один H1.
- [ ] Собрать страницу из компонентов дизайн-системы
- [ ] Проверка: все секции рендерятся; ровно один H1; внутренние ссылки ведут на будущие ЧПУ; адаптив 375/768/1280; `:focus-visible`; консоль чистая; JSON-LD структурно валиден
- [ ] **Чек-поинт:** показать Главную заказчику — это валидация языка на реальной странице. **Не продолжать к Фазе 2 без ОК.**

## Заметки по исполнению

- **Git:** проект не под git. Коммиты в плане опущены; если нужен контроль версий — предложу `git init` (чек-поинт = согласование, не коммит).
- **Проверка:** `static-python` (:8000) + DOM-чтение; скриншоты — когда панель починится, либо приватным Artifact.
- **Открытые входные:** финальный список отраслей; реальные цифры/отзывы/лого/фото (плейсхолдеры до замены); решение по продакшен-стеку.
- Фазы 2–3 детализируются посекционно при старте каждой; визуальный язык фиксируется на Фазе 1.
