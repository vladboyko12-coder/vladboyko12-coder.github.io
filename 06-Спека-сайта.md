**Дата:** 2026-07-22
**Основание:** [01-Аналитика-конкурентов.md](01-Аналитика-конкурентов.md), [05-Дизайн-концепт.md](05-Дизайн-концепт.md).
**Статус:** заменяет прежнюю рекомендацию (Макет-1) — визуальный язык и объём пересмотрены с заказчиком.

---

## Решения (зафиксировано)

- **Формат:** многостраничный **САЙТ** (не лендинг), с прицелом на органический трафик (**SEO**).
- **Визуальный язык — «Редакторский»:** тёплая бумага + чернила + моноширинный «голос», крупная типографика, строгая сетка, тонкие линии. Герой-тезис: «Порядок в продажах. За 30 дней.»
- **Акцент: БОРДО `#7E2434`** — единственный акцент.
- **Отвергнуто:** прежние 3 макета (шаблонно/дёшево), бело-синее болото, тёмная премиум-тема, **зелёный акцент**, «инфобизнес-оранж».

## 2. Дизайн-система

**Палитра** (осознанно один «бумажный» light-режим):
`ink #17150F` · `paper #F3EFE6` · `panel #ECE6D9` · `muted #6E675A` · `line rgba(23,21,15,.16)` · **`accent бордо #7E2434`**. Семантика форм (успех/ошибка) — отдельно от акцента.

**Типографика:**
- Display — плотный гротеск (прототип: системный стек; продакшен: заинлайнить лицензированный гротеск).
- Voice/utility — моно (Cascadia/Consolas): метки, метрики, номера.
- Body — системный sans. Чёткая шкала, трекинг, `tabular-nums` для цифр.

**Компоненты** (переиспользуются на всех страницах): шапка (лого IN—CRM, меню, телефон, CTA «Аудит»), подвал; кнопки ink / accent / ghost / текст-ссылка; карточки услуги, кейса, статьи, тарифа, отзыва, члена команды; нумерованный индекс, строка-статистика, FAQ-аккордеон, формы, блок аудита, хлебные крошки. Состояния фокуса, `prefers-reduced-motion`, адаптив (десктоп/планшет/моб + бургер).

## 3. Шаблоны страниц (состав)

- **Главная:** герой-тезис → цифры/лого → ключевые услуги → «как внедряем за 30 дней» → превью кейсов → превью тарифов → команда → отзывы → FAQ → аудит.
- **Услуги-хаб:** интро → сетка услуг → процесс → CTA.
- **Услуга-деталь:** что входит → как проходит → результат → цена/срок → связанные кейсы/отрасли → FAQ → форма. *(SEO: 1 услуга = 1 URL = 1 кластер запросов.)*
- **Отрасль:** боли отрасли → решение → релевантные кейсы → интеграции под отрасль → CTA. *(SEO под «amoCRM для <отрасль>».)*
- **Кейсы-список:** фильтр по отрасли/услуге + карточки «было → стало».
- **Кейс-деталь:** клиент → задача → решение → оцифрованный результат → отзыв.
- **Тарифы:** 3 тарифа + что входит + лицензии + FAQ + форма.
- **Блог-список:** рубрики + карточки статей.
- **Статья:** H1, оглавление, тело, автор, связанные статьи/услуги, CTA-аудит.
- **О компании / Команда:** экспертиза, лица, статусы партнёра, ценности.
- **Контакты:** форма, телефон/почта, карта, реквизиты, мессенджеры.

## 4. SEO-архитектура

- Семантический HTML, один H1/страница, корректная иерархия заголовков.
- Мета `title`/`description` на каждой странице; Open Graph / Twitter Card.
- Разметка Schema.org: Organization + **LocalBusiness (Калининград)**, Service, BreadcrumbList, Article, FAQPage, Review/AggregateRating.
- `sitemap.xml`, `robots.txt`, `canonical`, ЧПУ, внутренняя перелинковка (услуга ↔ отрасль ↔ кейс ↔ статья).
- Скорость (инлайн/оптимизация ассетов), адаптивность, доступность — как факторы ранжирования.
- Локальное SEO: Калининград + регионы присутствия (17 городов), консистентный NAP.
- Контент-кластеры: услуга-пилар + отрасли + кейсы + статьи вокруг неё.

## 5. Технический подход

- **Прототип / дизайн-предложение:** связанные самодостаточные HTML-страницы, общий CSS дизайн-системы.
- **Продакшен с блогом и SEO:** рекомендуется статический генератор (Astro / 11ty) или CMS — отдельное решение по реализации, вне дизайн-спека (ручной HTML не масштабируется на блог).
- **Шрифты:** для продакшена заинлайнить лицензированные display+mono; в прототипе — системные стеки.

## 6. Фазы сборки

1. **Дизайн-система + Главная** + сквозной каркас (шапка/подвал/навигация/блок аудита).
2. **Ключевые внутренние:** Услуги (хаб + деталь), Тарифы, Кейсы (список + деталь), О компании, Контакты.
3. **SEO-страницы:** Отрасли (хаб + шаблон), Блог (список + статья).
